# 1.2 Interaction and Gestures

> [!abstract] TL;DR
> In mobile development, web-like hover and active CSS states are replaced by touch responder components and gesture handler threads. High-performance interactions use declarative gesture detectors and animations that run entirely on the native UI thread to avoid the asynchronous bridge bottleneck.

## Digest

On the web, click events are simple and click targets are relatively large. On mobile, touch interactions are highly nuanced: users expect instant visual feedback, swipe capabilities, multi-touch pinch-to-zoom, and velocity-sensitive dismissals.

### Touch Primitives: Pressable vs. TouchableOpacity

React Native provides two primary ways to make components interactive:

1. **`TouchableOpacity`**: A legacy wrapper that automatically decreases the opacity of its child when pressed. It is simple but offers limited customization.
2. **`Pressable`**: The modern standard for touch feedback. It does not enforce any pre-packaged visual changes. Instead, it exposes a state hook through its `style` and `children` properties, letting you customize layout and styling dynamically when pressed:

```tsx
import { Pressable, Text, StyleSheet } from 'react-native';

export default function MyButton() {
  return (
    <Pressable
      style={({ pressed }) => [
        styles.button,
        pressed && styles.buttonPressed,
      ]}
    >
      <Text style={styles.text}>Submit</Text>
    </Pressable>
  );
}

const styles = StyleSheet.create({
  button: {
    backgroundColor: '#007AFF',
    padding: 12,
    borderRadius: 8,
  },
  buttonPressed: {
    backgroundColor: '#0051A8',
    opacity: 0.9,
  },
  text: {
    color: '#ffffff',
    textAlign: 'center',
  },
});
```

### The Async Bridge Bottleneck

To understand why custom gestures require third-party libraries, we must understand the React Native architecture. 

In standard React Native, when a user touches the screen:
1. The native OS registers coordinates.
2. Touch events are serialized and sent over the asynchronous bridge to the JavaScript thread.
3. React states change, recalculating styles.
4. Calculated styles are serialized and sent back over the bridge to the Native thread.
5. The Native thread updates the display.

If the JS thread is executing heavy code (e.g., parsing JSON, rendering components), this round-trip is delayed. This causes gestures to "lag" behind the user's finger.

### Declarative UI Interactions

To bypass the bridge bottleneck, modern React Native relies on two core libraries:

- **`react-native-gesture-handler`**: Bypasses the React Native touch responder system by defining gesture detectors that intercept and track touches directly on the Native thread.
- **`react-native-reanimated`**: Allows developers to write Javascript functions called **worklets** that compile into native code and run directly on a dedicated UI thread. Gesture coordinates are mapped to UI styles synchronously, resulting in 60fps/120fps animations.

Here is how you map a swipe gesture to a style value declaratively using Reanimated:

```tsx
import React from 'react';
import { StyleSheet } from 'react-native';
import { GestureDetector, Gesture } from 'react-native-gesture-handler';
import Animated, { useSharedValue, useAnimatedStyle, withSpring } from 'react-native-reanimated';

export default function SwipeableBox() {
  // 1. Declare a shared value (stored on the UI thread)
  const translateX = useSharedValue(0);

  // 2. Set up the gesture detector
  const panGesture = Gesture.Pan()
    .onUpdate((event) => {
      translateX.value = event.translationX;
    })
    .onEnd(() => {
      // Smoothly animate back to center using spring physics
      translateX.value = withSpring(0);
    });

  // 3. Define the animated style
  const animatedStyle = useAnimatedStyle(() => ({
    transform: [{ translateX: translateX.value }],
  }));

  // 4. Attach the detector and style to an Animated view
  return (
    <GestureDetector gesture={panGesture}>
      <Animated.View style={[styles.box, animatedStyle]} />
    </GestureDetector>
  );
}

const styles = StyleSheet.create({
  box: {
    width: 100,
    height: 100,
    backgroundColor: '#3b82f6',
    borderRadius: 8,
  },
});
```

## Drill

Create a horizontal swipe-to-dismiss card component.

### Task Description
1. Construct a list card containing mock item data.
2. Wrap the card inside a `GestureDetector` that listens for horizontal drag (Pan) gestures.
3. Translate the card's position smoothly in response to finger movement.
4. Add dismissal logic:
   - If the user swipes past a threshold (e.g., 40% of the screen width), animate the card completely off-screen and trigger a dismiss callback.
   - If the user releases before the threshold, use spring physics to return the card to its original position.
5. Ensure the animation runs on the native UI thread using `GestureDetector` and `react-native-reanimated`.

> [!example] Success criteria
> - [ ] The card tracking is fluid and lags zero frames behind the finger movement.
> - [ ] The component does not use the standard React Native `Animated` library; it must use `react-native-reanimated` (v3+) hooks (`useSharedValue`, `useAnimatedStyle`, `withSpring`, `withTiming`).
> - [ ] The drag threshold is calculated dynamically or configured clearly.
> - [ ] The card snaps back smoothly to center if released before the threshold.
> - [ ] Swiping past the threshold triggers an exit animation and calls a dismissal handler function.
