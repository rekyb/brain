# 4.3 Networking

> [!abstract] TL;DR
> Modern Android networking is Retrofit with `suspend` endpoint functions and `kotlinx.serialization` for JSON — declare the API shape, call it like any coroutine, and handle errors with `runCatching` or a `Result` wrapper in the repository.

## Digest

### From callbacks to coroutines

In 2021 a Retrofit endpoint returned `Call<T>` and you fired it with `enqueue(object : Callback<T> { onResponse(…); onFailure(…) })` — two callback methods, explicit thread checks, and manual LiveData posting. The modern path replaces this entirely: mark the interface function `suspend`, call it inside a coroutine, and receive the response body directly (or catch a thrown `HttpException` or `IOException`).

The Retrofit builder is largely unchanged; what you swap is the converter factory:

```kotlin
val retrofit = Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(Json.asConverterFactory("application/json".toMediaType()))
    .client(okHttpClient)
    .build()
```

The `Json.asConverterFactory` extension comes from the `retrofit2-kotlinx-serialization-converter` library, which routes serialisation to `kotlinx.serialization` instead of Gson.

### Defining the API interface

```kotlin
interface QuoteApi {
    @GET("quotes/random")
    suspend fun getRandomQuote(): QuoteResponse
}
```

That is all. Retrofit generates the implementation at runtime. Call it from the repository:

```kotlin
suspend fun fetchRandomQuote(): Result<Quote> = runCatching {
    quoteApi.getRandomQuote().toDomain()
}
```

`runCatching` wraps both the HTTP call and the deserialisation step in a `Result<T>`. The ViewModel maps `onSuccess` / `onFailure` to UI state — no uncaught exception propagates to a composable.

### Modeling responses with @Serializable

```kotlin
@Serializable
data class QuoteResponse(
    val content: String,
    val author: String,
)

fun QuoteResponse.toDomain() = Quote(text = content, author = author)
```

`@Serializable` generates a serialiser at compile time via KSP. Field names must match JSON keys, or use `@SerialName("json_key")` to remap them. There is no runtime reflection — serialisation is fast, safe under code shrinking (no ProGuard keep rules needed for model fields), and has no nullability surprises with Kotlin data classes the way Gson did.

### OkHttp logging interceptor

Attach `HttpLoggingInterceptor` to your OkHttp client during development:

```kotlin
val loggingInterceptor = HttpLoggingInterceptor().apply {
    level = HttpLoggingInterceptor.Level.BODY
}

val okHttpClient = OkHttpClient.Builder()
    .addInterceptor(loggingInterceptor)
    .build()
```

`Level.BODY` prints full request and response bodies in Logcat — invaluable when shaping a new endpoint. Set `level = NONE` for release builds (or conditionally omit the interceptor via `BuildConfig.DEBUG`).

### Error handling

A `suspend` Retrofit function throws `HttpException` for 4xx/5xx responses and `IOException` for network failures. Containing these in the repository keeps errors typed rather than propagating raw exceptions:

```kotlin
sealed interface NetworkResult<out T> {
    data class Success<T>(val data: T) : NetworkResult<T>
    data class Error(val code: Int?, val message: String?) : NetworkResult<Nothing>
    data object Loading : NetworkResult<Nothing>
}
```

The repository returns the sealed type; the ViewModel maps it to the appropriate `UiState` fields (`isLoading`, `error`, `data`). The composable remains exception-free.

### What changed since 2021

- **`suspend` functions replace `Call<T>` + `enqueue`.** Support for `suspend` in Retrofit is built-in since 2.6 — no `CoroutineCallAdapterFactory` needed. Remove all `enqueue(object : Callback<T> { … })` patterns.
- **`kotlinx.serialization` replaces Gson.** `GsonConverterFactory` is gone; the converter is `Json.asConverterFactory(…)` from `retrofit2-kotlinx-serialization-converter`. Models annotated with `@Serializable` require no `TypeToken`, no `GsonBuilder` configuration, and behave correctly with Kotlin non-null types.
- **Moshi** is another viable option but `kotlinx.serialization` is the Kotlin-first, KSP-driven choice that Jetpack now recommends for new projects.
- **Ktor Client** is a Kotlin Multiplatform-capable alternative to Retrofit. Worth knowing it exists; Retrofit is the right starting point here since you already know its mental model.

## Drill

Work in the capstone project.

**Part A — dependencies and OkHttp client**

Add Retrofit, the `retrofit2-kotlinx-serialization-converter`, `kotlinx.serialization` (plugin + runtime), and the OkHttp logging interceptor to your module's dependencies. Create an `OkHttpClient` with the logging interceptor attached (debug only if you prefer). Provide it via the Hilt network module.

**Part B — API interface and Retrofit instance**

Use any live, keyless public JSON API — e.g. `https://zenquotes.io/api/random` for a quote, or a placeholder like `https://jsonplaceholder.typicode.com/todos/1`; if one is down, pick another — the point is a real network round-trip into a `@Serializable` model. Define a `@Serializable` data class matching its response shape. Write a Retrofit interface with a single `suspend` endpoint for that route. Build the Retrofit instance in the Hilt module, using the `kotlinx.serialization` converter factory, and expose the API interface as a binding.

**Part C — repository and ViewModel**

Create a `QuoteRepository` (interface + implementation) that calls the Retrofit endpoint inside a `suspend` function and returns `Result<Quote>` via `runCatching`. In a ViewModel, expose a `UiState` with `quote: Quote?`, `isLoading: Boolean`, and `error: String?`. Trigger the fetch on a user action (e.g., a "Fetch quote" button) inside `viewModelScope`. Handle both success and failure branches in the UI — the app must not crash on a network error.

> [!example] Success criteria
> - [ ] A real quote from the chosen API renders in the UI on a successful fetch.
> - [ ] A network error (e.g., airplane mode on the emulator) shows an error message rather than crashing.
> - [ ] The Retrofit endpoint is declared as `suspend fun` — no `Call<T>`, `enqueue`, or `Callback` exists in the project.
> - [ ] The response model uses `@Serializable`; no `GsonConverterFactory` or Moshi appears in the project.
> - [ ] The OkHttp logging interceptor output is visible in Logcat during a successful fetch.
