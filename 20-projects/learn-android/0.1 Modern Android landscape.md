# 0.1 Modern Android landscape

> [!abstract] TL;DR
> In five years the entire Android UI stack flipped: Compose replaced XML as the default UI toolkit, coroutines+Flow replaced AsyncTask and largely LiveData, and the build system gained a type-safe version catalog. This chapter maps every shift so nothing surprises you during the rest of the path.

## Digest

The biggest change is the UI layer. In 2021 you wrote `activity_main.xml`, inflated views, set click listeners imperatively, and wired `LiveData` observers in `onCreateView`. That model still compiles but Google now labels it legacy. **Jetpack Compose** is the default for all new projects: you describe what the UI should look like given the current state, and the framework redraws only the parts that changed. The new "Empty Activity" template in Android Studio ships with zero XML layout files.

**Kotlin is the only officially supported language.** Java still compiles, but all documentation, libraries, and samples are Kotlin-first. Every API you will use in this path has idiomatic Kotlin surfaces designed around extension functions, lambdas, and coroutines.

The async story consolidated around coroutines. `AsyncTask` was deprecated in API 30 and removed from the SDK in API 33. RxJava usage in new projects has dropped sharply. The platform answer is **coroutines** (structured, cancellable, dispatcher-aware) paired with **Flow** (reactive streams that compose cleanly with coroutines). `LiveData` is not deprecated but its scope narrowed: ViewModels now expose `StateFlow` or `SharedFlow`, and `LiveData` is mainly kept alive for legacy or inter-process contexts.

**Material 3** (also called Material You) replaced Material 2. It introduced dynamic color — on Android 12+ the system derives a color scheme from the user's wallpaper — and a revised component set. Most Material 2 Compose component names don't exist in the `androidx.compose.material3` package; you will get used to the new names quickly.

The build system modernized significantly. Groovy `build.gradle` files have a full Kotlin DSL equivalent (`build.gradle.kts`), which gives autocomplete and compile-time type checking. More importantly, new projects are scaffolded with a **version catalog** (`gradle/libs.versions.toml`): one place to declare every dependency version, with typed accessors like `libs.compose.ui`. This replaces the old practice of scattering version strings across `ext {}` blocks in the root `build.gradle`.

### What changed since 2021

- **UI toolkit:** Compose is the default; XML/Views/Fragments are legacy. The "Empty Activity" template generates Compose, not XML. No new Google-published API guidance targets XML-first development.
- **Async:** `AsyncTask` removed (API 33+). Coroutines + Flow are the platform standard. `LiveData` narrowed in scope — ViewModels expose `StateFlow` now.
- **Material Design:** Material 3 / Material You with dynamic color (Android 12+). The M2 Compose package (`material`) and M3 (`material3`) coexist but are separate; M2 components don't exist in M3.
- **Notifications:** Android 13 added `POST_NOTIFICATIONS` as a **runtime** permission. Apps targeting API 33+ must request it explicitly before posting; silent delivery no longer works without the grant.
- **Edge-to-edge:** Enforced by default from Android 15. Content draws behind system bars; you handle insets with `WindowInsets` APIs.
- **Predictive back:** Android 13+ introduced the predictive-back gesture; Android 14+ enforces `OnBackPressedCallback`. The `onBackPressed()` override is deprecated.
- **Background work:** Stricter limits on background and foreground services. `WorkManager` is the recommended API for deferrable guaranteed work.
- **Gradle:** Kotlin DSL (`build.gradle.kts`) + version catalogs (`libs.versions.toml`) are the standard. Groovy still works but new templates use `.kts`.
- **Compose compiler:** From Kotlin 2.0, the Compose compiler ships as a Kotlin compiler plugin via `kotlin("plugin.compose")` in Gradle — no separate `kotlinCompilerExtensionVersion` string required.

## Drill

Install the latest stable **Android Studio** (Meerkat or newer as of mid-2026). Create a fresh project using the **Empty Activity** template — this now generates Compose by default; there is no separate "Empty Compose Activity" option.

Once the project is generated, open each file below and write a brief annotation or comment next to any element that is new to you since 2021. You can annotate in a scratch Markdown note or directly as inline code comments.

1. `app/src/main/java/.../MainActivity.kt` — Find `setContent {}`, `@Composable`, the generated composable function, `Surface`, and `MaterialTheme`. Note: there is no `setContentView(R.layout....)`.
2. `app/build.gradle.kts` — Find the `plugins {}` block (especially the compose plugin entry), and `dependencies {}` using `libs.*` version-catalog accessors.
3. `gradle/libs.versions.toml` — The version catalog. Locate the `[versions]`, `[libraries]`, and `[plugins]` sections. Note how versions and library declarations are separated.
4. `app/src/main/java/.../ui/theme/Theme.kt` — How theming moved from XML `styles.xml` attributes to Kotlin `MaterialTheme` with `ColorScheme`.
5. `app/src/main/java/.../ui/theme/Color.kt` and `Type.kt` — These replaced `res/values/colors.xml` and `res/values/styles.xml`.

Run the app on an emulator (API 34+ recommended). It should display a "Hello Android!" or similar greeting.

> [!example] Success criteria
> - [ ] Android Studio installed; project created from the Empty Activity template.
> - [ ] App builds and runs on an emulator (API 34+ recommended).
> - [ ] You have written annotations on what each of the five listed files does.
> - [ ] You have identified at least five things that are new or different versus your 2021 memory.
> - [ ] You can articulate in one sentence what `setContent {}` is replacing and why.
