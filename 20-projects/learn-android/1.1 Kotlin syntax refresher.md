# 1.1 Kotlin syntax refresher

> [!abstract] TL;DR
> A rapid reactivation of Kotlin's everyday syntax — null safety, expression-form control flow, and smart casts — through eight short functions you implement by hand.

## Digest

If you wrote Kotlin in 2021, the language hasn't changed dramatically at the syntax level. The thing that trips returning developers is forgetting how much of Kotlin is *expression-based* and defaulting back into Java-shaped imperative style.

**Null safety** is the most important thing to get sharp again. The type system distinguishes `String` (non-null, compiler-guaranteed) from `String?` (nullable). Operators let you handle the nullable case without explicit null checks:

```kotlin
val length: Int = name?.length ?: 0       // safe call + Elvis operator
val trimmed: String = name?.trim() ?: return  // early-return with Elvis
```

`?.let { }` runs a block only when the value is non-null:

```kotlin
user?.let { u -> sendWelcome(u.email) }
```

Avoid `!!` entirely in production code — it converts a nullable into a guaranteed `NullPointerException`. Its only acceptable use is in tests where you want the crash to signal a broken assumption.

**`when` is an expression**, not just a statement. It replaces both `switch` and multi-branch `if-else`, and it can return a value:

```kotlin
val label = when (status) {
    Status.ACTIVE -> "Active"
    Status.PAUSED -> "Paused"
    Status.DONE   -> "Done"
}   // no `else` needed — compiler verifies exhaustiveness on enums/sealed types
```

**Smart casts** let the compiler track that you have already checked a type, so you can use the narrowed type without an explicit cast:

```kotlin
fun describe(value: Any?): String {
    if (value is String) return "string of length ${value.length}"   // smart-cast to String
    if (value is Int)    return "int: $value"
    return "unknown"
}
```

**Expression bodies** drop the braces and `return` for single-expression functions:

```kotlin
fun double(n: Int): Int = n * 2
fun greet(name: String?): String = "Hello, ${name ?: "stranger"}!"
```

These are style, not requirements — but idiomatic Kotlin uses them heavily, and you'll see them throughout the rest of this path.

## Drill

Work in **Kotlin Playground** (play.kotlinlang.org) or a scratch `.kt` file — no Android project needed yet.

Implement each function below. The signatures are fixed; write only the bodies. Do not use `!!` anywhere.

```kotlin
// 1. Parse a String to Int; return `default` if the string is not a valid integer.
fun safeParseInt(s: String?, default: Int): Int = TODO()

// 2. FizzBuzz for a single number using `when` as an expression.
//    Returns "FizzBuzz" (divisible by both), "Fizz" (by 3), "Buzz" (by 5),
//    or the number as a string otherwise.
fun fizzBuzz(n: Int): String = TODO()

// 3. Join a list of nullable strings, skipping nulls, with the given separator.
//    nullSafeJoin(listOf("a", null, "b"), " | ") == "a | b"
fun nullSafeJoin(parts: List<String?>, sep: String): String = TODO()

// 4. Return the longest string in the list, or null if the list is empty.
fun longest(items: List<String>): String? = TODO()

// 5. Given a nullable Int, return its square if non-null and positive, otherwise 0.
fun squarePositive(n: Int?): Int = TODO()

// 6. Classify an Int as "negative", "zero", or "positive" using `when` as an expression.
fun classify(n: Int): String = TODO()

// 7. Safely return the first element of a nullable list, or the provided default.
fun <T> firstOrDefault(list: List<T>?, default: T): T = TODO()

// 8. Return a string description of any value using smart cast:
//    - String  → "string of length ${it.length}"
//    - Int     → "int: $it"
//    - anything else (including null) → "unknown"
fun describe(value: Any?): String = TODO()
```

Write a `main()` that calls each function with at least one representative input and prints the result so you can verify correctness.

> [!example] Success criteria
> - [ ] All 8 functions compile and run in Kotlin Playground or a scratch file.
> - [ ] `main()` prints verifiable output for each function.
> - [ ] `!!` does not appear anywhere in your implementations.
> - [ ] At least four functions use an expression body (`fun f(): T = ...`).
> - [ ] `fizzBuzz` and `classify` use `when` as an expression (assigned to a value), not as an `if-else` chain.
> - [ ] `safeParseInt` handles null input without crashing.
