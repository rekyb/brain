# 2.5 Material 3 and theming

> [!abstract] TL;DR
> Material 3 (`androidx.compose.material3`) is the current design system for Compose: a `MaterialTheme` wrapping composable provides colour, typography, and shape tokens that propagate through the tree, with built-in dark-mode support and Android 12+ dynamic colour.

## Digest

### Material 3 vs Material 2

If you used Material Design components in 2021, you used Material 2 (`com.google.android.material` / `androidx.compose.material`). Material 3 (also called Material You) is the successor, with:

- A richer, role-based colour scheme (primary, secondary, tertiary, surface, error — each with `On*` and `Container`/`OnContainer` variants).
- Updated components (`Button`, `Card`, `TopAppBar`, `NavigationBar`, `FloatingActionButton`, etc.) with refreshed shapes and motion.
- **Dynamic colour** — on Android 12+, the colour scheme derives from the user's wallpaper automatically.

The import path is `androidx.compose.material3.*`. Do not mix `material` and `material3` imports in the same project — the component APIs differ and they conflict visually.

### MaterialTheme

`MaterialTheme` is a `CompositionLocal` provider that makes colour, typography, and shape available to any composable in the tree without explicit parameter threading:

```kotlin
MaterialTheme(
    colorScheme = myColorScheme,
    typography = myTypography,
    shapes = myShapes
) {
    // every composable inside reads from MaterialTheme.*
}
```

Inside any composable, access tokens via:
- `MaterialTheme.colorScheme.primary`
- `MaterialTheme.typography.headlineMedium`
- `MaterialTheme.shapes.medium`

### Defining a colour scheme

The Material 3 colour builder at `m3.material.io` generates a `ColorScheme` from a seed colour. Paste the generated values into a Kotlin file:

```kotlin
private val LightColorScheme = lightColorScheme(
    primary = Color(0xFF6650A4),
    onPrimary = Color(0xFFFFFFFF),
    primaryContainer = Color(0xFFEADDFF),
    // ... (remaining roles)
)

private val DarkColorScheme = darkColorScheme(
    primary = Color(0xFFD0BCFF),
    onPrimary = Color(0xFF381E72),
    // ...
)
```

`lightColorScheme` and `darkColorScheme` are convenience constructors with sensible defaults for roles you don't specify.

### Dark theme

Wire dark-mode by checking `isSystemInDarkTheme()` in your app-level theme composable:

```kotlin
@Composable
fun AppTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    dynamicColor: Boolean = true,
    content: @Composable () -> Unit
) {
    val colorScheme = when {
        dynamicColor && Build.VERSION.SDK_INT >= Build.VERSION_CODES.S -> {
            val context = LocalContext.current
            if (darkTheme) dynamicDarkColorScheme(context)
            else dynamicLightColorScheme(context)
        }
        darkTheme -> DarkColorScheme
        else -> LightColorScheme
    }

    MaterialTheme(colorScheme = colorScheme, typography = AppTypography, content = content)
}
```

### Dynamic colour (Material You)

On Android 12+ (API 31+), `dynamicLightColorScheme(context)` and `dynamicDarkColorScheme(context)` derive a full colour scheme from the user's wallpaper. This is the default behaviour in new Android Studio project templates and is what "Material You" means in practice. On API < 31, fall back to your static scheme.

Test dynamic colour on an emulator by changing the wallpaper in Settings — the app immediately restyles if you've wired it correctly.

### Typography

`Typography` holds named text styles. Override only what you need:

```kotlin
val AppTypography = Typography(
    headlineLarge = TextStyle(
        fontFamily = FontFamily.Default,
        fontWeight = FontWeight.Bold,
        fontSize = 32.sp
    ),
    bodyMedium = TextStyle(fontSize = 14.sp, lineHeight = 20.sp)
    // unspecified roles get Material 3 defaults
)
```

Reference in composables as `MaterialTheme.typography.headlineLarge`.

### What changed since 2021

- **Material 3 replaces Material 2.** The XML component library (`com.google.android.material`) still exists for View-based apps but is irrelevant in a pure-Compose project.
- **Dynamic colour is new.** There was no wallpaper-driven theming in Material 2; it's API 31+ only.
- **Colour roles replaced the primary/accent model.** The old `colorPrimary`/`colorAccent`/`colorPrimaryDark` attributes map roughly to `primary`, `secondary`, and `primaryContainer`, but the full M3 scheme has ~30 roles.
- **`AppCompat` theme is no longer the root.** Compose apps use `Theme.Material3.DayNight.NoActionBar` (or similar) as the Activity theme, or configure the window manually.

## Drill

Work in the Compose project. Create a dedicated `ui/theme/` package with `Theme.kt`, `Color.kt`, and `Type.kt`.

**Part A — custom colour scheme and typography**

Pick a seed colour and generate a Material 3 colour scheme (use the [Material Theme Builder](https://m3.material.io/theme-builder) or define values manually). Define both a light and dark `ColorScheme`. Define a custom `Typography` that overrides at least `headlineMedium` and `labelSmall` with non-default values (different size or weight).

**Part B — dark mode toggle**

Add a boolean state at the root of your app that overrides `isSystemInDarkTheme()`. Pass it into your `AppTheme`. Add a `Switch` in your main screen that toggles between light and dark mode — verify the entire app restyles.

**Part C — dynamic colour**

Enable `dynamicColor = true` in `AppTheme` with the API-level guard shown above. Run on an API 31+ emulator, change the wallpaper, and observe the app's colours shift to match.

> [!example] Success criteria
> - [ ] `AppTheme` is the single theming wrapper; all composables reference `MaterialTheme.*` tokens, not hardcoded `Color.*` values.
> - [ ] The light/dark toggle (Part B) visibly changes the colour scheme of all themed components instantly.
> - [ ] Custom `Typography` overrides are visible: at least one `Text` in the app uses `headlineMedium` and renders with the custom size/weight.
> - [ ] On an API 31+ emulator with a coloured wallpaper, dynamic colour is visibly active — the app's palette differs from the static fallback (Part C).
> - [ ] No `@ColorRes`, `R.color.*`, or hardcoded hex strings outside `Color.kt`.
