# 2.4 Lazy lists

> [!abstract] TL;DR
> `LazyColumn`, `LazyRow`, and `LazyVerticalGrid` replace `RecyclerView` entirely — no adapter, no `ViewHolder`, no `DiffUtil` boilerplate — while composing items lazily so only visible ones are in memory.

## Digest

### LazyColumn and LazyRow

A `LazyColumn` measures and places only the items currently visible on screen, plus a small buffer. Items outside the window are discarded and recomposed when they scroll back into view. This gives the same recycling behaviour as `RecyclerView` without any infrastructure classes.

```kotlin
@Composable
fun HabitList(habits: List<Habit>) {
    LazyColumn(
        contentPadding = PaddingValues(horizontal = 16.dp, vertical = 8.dp),
        verticalArrangement = Arrangement.spacedBy(8.dp)
    ) {
        items(habits, key = { it.id }) { habit ->
            HabitItem(habit)
        }
    }
}
```

`LazyRow` is identical but horizontal. Both accept a `contentPadding` parameter — this is the padding *inside* the scroll area, so the first and last items can be padded without clipping the scrollbar or overscroll effect.

### The items DSL

Inside the `LazyListScope` lambda, you have:

- `items(list, key = { ... }) { item -> }` — for a `List<T>`.
- `itemsIndexed(list) { index, item -> }` — when you need the index.
- `item { }` — a single item (useful for headers, footers, loading indicators).
- `items(count) { index -> }` — integer-ranged items without a list.

These can be mixed freely in one `LazyColumn` to produce heterogeneous layouts (header + list + footer, sticky sections, etc.).

### The key parameter

The `key` lambda maps each item to a stable, unique identifier. Without it, Compose uses the list index — which means reordering or inserting items causes unnecessary recomposition and breaks animations and scroll-position memory. Always set `key` when items have identity:

```kotlin
items(habits, key = { habit -> habit.id }) { habit ->
    HabitItem(habit = habit)
}
```

Keys must be stable types that can survive process death (primitives, strings, `Parcelable`). A `UUID` stored as a `String` works; a class instance does not.

### LazyVerticalGrid

For grid layouts, `LazyVerticalGrid` accepts a `columns` parameter:

```kotlin
LazyVerticalGrid(
    columns = GridCells.Adaptive(minSize = 160.dp),  // as many columns as fit at ≥160 dp each
    contentPadding = PaddingValues(8.dp)
) {
    items(items, key = { it.id }) { item ->
        GridItem(item)
    }
}
```

`GridCells.Fixed(n)` gives a fixed column count; `GridCells.Adaptive(minSize)` fills the width responsively. `LazyHorizontalGrid` exists for the horizontal direction.

### What changed since 2021

- **`RecyclerView` + `Adapter` + `ViewHolder` + `DiffUtil` are gone.** All of that infrastructure collapses into a `LazyColumn` with an `items { }` block and a `key` lambda. The framework handles diffing internally.
- **No `notifyDataSetChanged`.** State-driven recomposition handles updates. Change the list in your `StateFlow`, and `LazyColumn` redraws exactly the items that changed.
- **No XML item layouts.** Each item is a composable function, composable with the same toolkit as any other UI.
- **`itemsIndexed` replaces position-based adapter patterns** where you needed the index for striping or headers.

## Drill

Work in the Compose project.

**Task:** Create a data class `ListItem(val id: Int, val title: String, val description: String)`. Generate 50 instances in a `remember { }` block (or a top-level `val`) with unique `id` values.

Display them in a `LazyColumn` using a custom `ListItemCard` composable for each row. The card should show the title prominently and the description in a secondary style, with a `Card` surface (elevation, rounded corners).

Stable `key`s must be set. The list must scroll smoothly on an emulator. Add a header item (a single `item { }` block) above the list that says "All items (50)".

> [!example] Success criteria
> - [ ] All 50 items are reachable by scrolling; the list does not freeze or stutter on the emulator.
> - [ ] `key = { it.id }` is set on the `items { }` call.
> - [ ] A header item ("All items (50)") renders above the first list item and scrolls with the list.
> - [ ] No `RecyclerView`, `Adapter`, `ViewHolder`, or XML item layout exists in the project.
> - [ ] Each `ListItemCard` is a standalone `@Composable` function — not an inner class or anonymous lambda with heavy logic.
